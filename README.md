# 思考プロセス フロー化ツール (ucf_flow)

熟練者の Q&A・故障対応の履歴から「切り分けの思考プロセス」を読み取り、分岐するフローチャートとして可視化・編集できる Web ツールです。2 つの UI バリエーションを収録しています。

| ツール | 説明 |
|--------|------|
| **A・パイプライン版** | 青基調・サイドバー付きの実務向け UI。PDF / テキスト / CSV 取り込み、ワークスペース管理、ナレッジDB を備えたフル機能版 |
| **スケッチ版** | 紙とペンの手書き風 UI。発想・整理にフォーカスした軽量版 |

---

## すぐ使う（単体 HTML）

`standalone/` の HTML はビルド済みで、**ファイルをブラウザで開くだけ**で動きます（ランタイム・フォント・依存物すべて同梱）。

- `standalone/pipeline-tool.html` … A・パイプライン版
- `standalone/sketch-tool.html` … スケッチ版

> インターネット接続があるとフォントが綺麗に表示されます（オフラインでも動作します）。

### AI 機能について

フロー自動生成などの AI 機能は、画面右上の **「API設定」** から OpenAI または Azure OpenAI の API キーを入力すると有効になります。キーはブラウザ内（このページ）でのみ使用され、サーバーには送信されません。

## ソースから動かす / 編集する

`src/` に編集用のソースを置いています。

```
src/
├─ support.js                         # 描画ランタイム（編集不要）
├─ フロー化ツール（A・パイプライン）.dc.html
├─ 思考プロセス フロー化ツール.dc.html
└─ assets/
   └─ sample-manual.pdf               # 取り込みデモ用サンプル
```

`.dc.html` は `support.js` を相対参照します。ローカルサーバ経由で開いてください（`file://` 直開きだと一部機能が制限されます）。

フロントエンドとバックエンドをまとめて起動する場合:

```bash
npm install
npm start
# Frontend: http://127.0.0.1:8000  （A・パイプライン版が開きます）
# Backend:  http://127.0.0.1:3001
```

### ログイン / 検証用アカウント

`npm start` でローカル起動するとログイン画面が表示されます。開発・検証用に以下のアカウントが自動作成されます。

| 種別 | メールアドレス | パスワード |
|------|----------------|------------|
| 管理者 | `admin@example.test` | `Password123!` |
| 一般ユーザ | `user@example.test` | `Password123!` |

ユーザ情報とセッションは `data/auth-db.json` に保存されます。このファイルはローカルデータとして `.gitignore` 対象です。

検証用アカウントの自動作成は `127.0.0.1` / `localhost` / `::1` で起動した場合だけ既定で有効です。明示的に切り替える場合:

```bash
AUTH_SEED_DEMO=0 npm start
# または
AUTH_SEED_DEMO=1 npm start
```

管理者でログインすると、サイドバーの **ユーザ管理** からユーザ作成、ロール変更、無効化、有効化、パスワード再設定ができます。

ポートを変える場合:

```bash
FRONTEND_PORT=8080 BACKEND_PORT=3002 npm start
```

フロントエンドは起動時に `/app-config.js` からバックエンド URL を受け取るため、`BACKEND_PORT` を変えてもログイン API に接続できます。別ホストやリバースプロキシ経由で動かす場合は `BACKEND_PUBLIC_URL` を指定してください。

個別起動もできます。

```bash
npm run frontend
npm run backend
```

バックエンドは `/api/health` で起動確認できます。

従来どおり Python の簡易サーバでもフロントだけを開けます。

```bash
cd src
python -m http.server 8000
# → http://localhost:8000/フロー化ツール（A・パイプライン）.dc.html
```

編集後は同じ HTML をそのままブラウザで再読み込みすれば反映されます。

## 実装時の約束

- 表示崩れ・フロー線・分岐などの修正は、想像で調整せず、DOM 座標・SVG path・テストケースなどの事実を確認してから行う。
- 修正後は、再発しやすいケースを `scripts/verify-flow-search.mjs` などの検証に追加し、ブラウザ上でも重なりや接続状態を確認する。
- ノードとエッジの視覚的な問題は、ノード矩形・ラベル矩形・エッジ経路を分けて検証し、どれが原因かを明確にしてから直す。
